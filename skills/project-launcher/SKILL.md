# Project Launcher Skill - PRD → 개발 연결 스킬

이 스킬은 **prd-workflow-v2**에서 완성된 PRD를 검증하고, **supervisor-report-v2**로 개발을 시작하는 전체 흐름을 자동화합니다.

---

## 핵심 원칙

> **"검증 없이 시작하지 마라."**

PRD와 기술 명세가 완전한지 확인한 후에만 개발을 시작합니다.

> **"모호한 시작 금지."**

"프로젝트 시작" 대신 구체적인 FR 단위로 개발을 지시합니다.

> **"상태를 항상 추적하라."**

프로젝트 전체 진행 상황을 한 곳에서 관리합니다.

---

## 사용 시점

| 상황 | 사용 |
|------|------|
| prd-workflow-v2 완료 후 개발 시작 | ✅ |
| 기존 PRD로 새 개발 시작 | ✅ |
| 개발 중단 후 재개 | ✅ |
| PRD 없이 개발 시작 | ❌ (prd-workflow-v2 먼저) |

---

## 워크플로우 개요

```
Phase 1: PRD 완성 검증
    ↓ 실패 시 → prd-workflow-v2로 복귀
Phase 2: 기술 명세 검증
    ↓ 실패 시 → 기술 명세 재생성
Phase 3: 개발 계획 수립
    ↓
Phase 4: 프로젝트 상태 파일 생성
    ↓
Phase 5: 개발 시작 (supervisor-report-v2 호출)
    ↓
Phase 6: 진행 상황 모니터링
```

## Phase 0: MCP 설정 확인 (최초 1회)

> 프로젝트 시작 전 필요한 MCP가 설정되어 있는지 확인합니다.
> MCP는 시스템 레벨 설정이므로 한 번만 하면 됩니다.

### 0.1 필수 MCP 목록

| MCP | 용도 | 필수도 |
|-----|------|--------|
| `context7` | 최신 라이브러리 문서 | ⭐⭐⭐ 필수 |
| `filesystem` | 파일 시스템 접근 | ⭐⭐⭐ 필수 |
| `git` | 버전 관리 | ⭐⭐⭐ 필수 |
| `memory` | 장기 기억 | ⭐⭐ 권장 |

### 0.2 프로젝트별 추가 MCP

| 프로젝트 유형 | 추가 MCP |
|---------------|----------|
| PostgreSQL 사용 | `postgres` |
| MongoDB 사용 | `mongodb` |
| GitHub 연동 | `github` |
| Notion 문서 연동 | `notion` |

### 0.3 MCP 설정 파일 생성

프로젝트 루트에 `.claude/mcp_config.json` 템플릿 생성:

```json
{
  "mcpServers": {
    "context7": {
      "command": "npx",
      "args": ["-y", "@anthropics/context7-mcp"]
    },
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "."]
    },
    "git": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-git"]
    },
    "memory": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-memory"]
    }
  }
}
```

### 0.4 MCP 미설정 시 안내

```markdown
⚠️ MCP 설정 필요

이 프로젝트에서 다음 MCP가 필요합니다:

**필수**:
- context7 (최신 라이브러리 문서)
- filesystem (파일 접근)
- git (버전 관리)

**권장**:
- memory (세션 간 기억)

**설정 방법**:
1. `~/.config/claude/claude_desktop_config.json` 파일 열기
2. 위 MCP 설정 추가
3. Claude Code 재시작

또는 프로젝트별 설정:
1. `.claude/mcp_config.json` 파일 확인
2. 필요한 MCP 활성화
```

### 0.5 MCP 설정 확인 체크리스트

| # | 확인 항목 | 상태 |
|---|-----------|------|
| 1 | context7 사용 가능? | ✅/❌ |
| 2 | filesystem 사용 가능? | ✅/❌ |
| 3 | git 사용 가능? | ✅/❌ |
| 4 | 프로젝트별 추가 MCP? | ✅/❌/해당없음 |

> **참고**: MCP는 시스템 레벨 설정입니다. 스킬/에이전트가 자동으로 설치할 수 없습니다.
> 위 안내를 따라 사용자가 직접 설정해야 합니다.

## Phase 1: PRD 완성 검증

### 1.2 CLAUDE.md 검증

CLAUDE.md가 없거나 불완전한 경우 자동 생성합니다.

#### 필수 섹션 체크

| # | 섹션 | 필수 |
|---|------|------|
| 1 | 프로젝트 개요 | ✅ |
| 2 | 기술 스택 | ✅ |
| 3 | 현재 진행 상황 | ✅ |
| 4 | 중요 파일 위치 | ✅ |
| 5 | **세션 규칙** | ✅ 필수! |

#### CLAUDE.md 없는 경우

prd-workflow-v2의 6.4 템플릿을 사용하여 자동 생성:
- PRD-FINAL.md에서 프로젝트 정보 추출
- conventions.md에서 기술 스택 추출
- current-state.md에서 현재 상태 추출
- 세션 규칙 섹션 자동 포함

### 1.3 PRD 품질 확인

```markdown
## PRD 검증 체크리스트

### 필수 섹션 존재 여부

| # | 섹션 | 존재 | 내용 충분 |
```
